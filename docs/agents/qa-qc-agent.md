# QA/QC Agent

- **소속**: Cross-Team

---

## 1. 역할 목적

팀 경계를 넘는 E2E 시나리오를 검증합니다.  
단위 테스트보다는 실제 사용자 흐름(세션 생성 → SMS → AI 대화 → 영상 재생) 전체의 정합성을 검증합니다.

---

## 2. 담당 범위

| 분류 | 내용 |
|---|---|
| E2E 시나리오 | 정상 흐름 전체 검증 |
| 엣지 케이스 | SMS 실패, WS 단절, STT 오류, Cast 연결 실패 등 |

## 4. E2E 테스트 시나리오

### 정상 흐름
```
1. 관리자 로그인 → userCode 생성 → 영상 업로드 → 질문 등록
2. Flutter 앱에서 userCode 입력 + 전화번호 입력 → 세션 생성
3. Chromecast 연결 → TV에 AI 아바타 표시
4. "초대 SMS 발송" → 실제 SMS 수신 확인
5. User B가 SMS 링크 접속 → AI 대화 시작
6. 4개 질문 완료 → 영상 재생 트리거
7. TV에서 DB 기반 영상 재생 확인
```

### 엣지 케이스
| 시나리오 | 기대 동작 |
|---|---|
| SMS 발송 실패 (Solapi 오류) | 오류 메시지 표시, 재시도 가능 |
| WebSocket 연결 단절 | 재연결 시도 또는 연결 끊김 UI 표시 |
| STT 음성 인식 실패 | 텍스트 입력 폴백 활성화 |
| Chromecast 연결 실패 | 로컬 영상 재생 폴백 |
| proposal-data 조회 실패 | 기본 QUESTIONS로 폴백, 오류 로그 |
| userCode 없는 세션 생성 | 기본 질문 사용 |
| User B 중복 접속 | 두 번째 접속 무시 |
